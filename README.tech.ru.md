## О себе

Я практикующий DevOps/DevSecOps/SRE-инженер с 11+ годами опыта в Linux-инфраструктуре, высоконагруженной эксплуатации и диагностике проблем в продакшене.

Совмещаю инфраструктурную ответственность с серверной разработкой, особенно на стыке надежности платформы и прикладной логики продукта.

## Чем занимаюсь

- развиваю и сопровождаю боевую инфраструктуру с фокусом на SLA/SLO;
- автоматизирую диагностику, смягчение и восстановление после инцидентов;
- проектирую CI/CD и GitOps-процессы для безопасных выкаток;
- строю наблюдаемость (метрики, логи, алерты) с упором на практическую полезность;
- внедряю усиление защищенности и DevSecOps-контроли в эксплуатационный контур.

## Опыт разработки на Go и PHP

### Go (серверная часть, воркеры, Telegram-интеграции)

Промышленная разработка на Go в многосервисном Telegram-продукте:
- декомпозиция на `bot-api`, `app-core`, `worker-scheduler`, `worker-notify`, `worker-export`;
- внутренние API-границы с защитой сервисным токеном;
- паттерны идемпотентности (защита от дублей обновлений через `processed_updates`, устойчивые к конфликтам вставки);
- асинхронный экспорт (`csv/json/ics`) с загрузкой в S3 и временными ссылками на скачивание;
- встроенная в доменную логику аналитика и реферальные сценарии.

Проект: приватный репозиторий Go Telegram-бота (Go 1.22, Chi, pgx, AWS SDK v2).

### PHP (операционные платформы и API-сервисы)

Практическая серверная разработка на PHP в инфраструктурных продуктах:
- **GitOps Traefik Panel** (приватный репозиторий): слойная архитектура (`Action` + `Service` + `DTO`), проверка DNS/WHOIS, автоматизация запросов на слияние в GitHub, опрос статусов выкладки, защитные механизмы (CSRF, ограничение частоты запросов, remember-token);
- **Внутренний API хранения статических файлов** (приватный проект): API загрузки/копирования/удаления файлов, корреляция по идентификатору запроса, нормализация и валидация путей, дедупликация по MD5 через жесткие ссылки, хранение метаданных в MySQL, контейнерный запуск с усилением защищенности.

- асинхронные воркеры для долгих операций и предсказуемой задержки бота;
